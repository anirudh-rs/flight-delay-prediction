# ✈️ US Flight Delay Prediction System

A full end-to-end data science project that predicts US domestic flight delays using historical airline data, real weather conditions, and machine learning. Built with Python, PostgreSQL, XGBoost, SHAP, and Tableau Public.

## 📁 Data Sources

Download the following files and place them in `data/raw/` before running the pipeline.

### 1. Flight Delay & Cancellation Dataset (2019–2023)
- **Source:** Kaggle — [Flight Delay and Cancellation Dataset 2019–2023](https://www.kaggle.com/datasets/patrickzel/flight-delay-and-cancellation-dataset-2019-2023)
- **File needed:** `flights_sample_3m.csv`
- **Place in:** `data/raw/FlightDelays/flights_sample_3m.csv`
- **Note:** Free Kaggle account required to download

### 2. Airport Database
- **Source:** OurAirports — [Direct Download](https://ourairports.com/data/airports.csv)
- **File needed:** `airports.csv`
- **Place in:** `data/raw/airports.csv`
- **Note:** No account required

### 3. Weather Data
- **Source:** Open-Meteo Historical Weather API
- **Note:** Fetched automatically by `src/05_fetch_weather.py` — no account or API key required
- Covers top 150 US airports across the full date range

## ⚙️ Setup & Reproduction

### Prerequisites
- Python 3.11+
- PostgreSQL 16
- Anaconda/conda

### 1. Clone the repo
```bash
git clone https://github.com/yourusername/flight-delay-predictor.git
cd flight-delay-predictor
```

### 2. Create conda environment
```bash
conda create -n flight_delay python=3.11 -y
conda activate flight_delay
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Configure database connection
Open `src/config.py` and update your PostgreSQL credentials:
```python
_password = "your_postgres_password"
```

### 5. Set up PostgreSQL
Create the database and run the schema:
```bash
psql -U postgres -c "CREATE DATABASE flight_delays;"
psql -U postgres -d flight_delays -f sql/create_tables.sql
```

### 6. Run the pipeline in order
```bash
python src/01_clean_data.py
python src/02_load_to_postgres.py
python src/03_feature_engineering.py
python src/05_fetch_weather.py
python src/04_train_model.py
python src/05_shap_analysis.py
python src/06_export_for_tableau.py
```

Pipeline output will be saved to `data/processed/`, `models/`, and `exports/`.

## 🤖 Models

Two XGBoost models are trained on 1.86 million flights across 2019, 2022, and 2023 (pandemic years excluded).

| Model | Type | Metric | Score |
|---|---|---|---|
| Binary Classifier | Will the flight be delayed 15+ mins? | ROC-AUC | 0.7203 |
| Bracket Classifier | Which delay bracket? | Weighted F1 | 0.5950 |

### Delay Brackets
| Bracket | Definition |
|---|---|
| On Time | Less than 15 minutes |
| Short Delay | 15–45 minutes |
| Medium Delay | 45–120 minutes |
| Long Delay | 120+ minutes |

## 🔍 Key Findings (SHAP Feature Importance)

The top factors driving flight delay predictions, ranked by mean absolute SHAP value:

1. **Departure Hour** — Evening flights are significantly more likely to be delayed due to schedule propagation throughout the day
2. **Route Delay Rate** — Some routes are chronically delayed regardless of conditions
3. **Airline Delay Rate** — Carrier reliability is a stronger predictor than weather
4. **Month** — Summer and holiday months drive the highest delay rates
5. **Day of Week** — Fridays show the highest delay rates consistently
6. **Cloud Cover** — Weather signal confirmed despite partial coverage
